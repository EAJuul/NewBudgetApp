## Public API to produce
```dart
// lib/features/categories/domain/category_group.dart
import 'package:freezed_annotation/freezed_annotation.dart';
import 'package:new_budget_app/domain/enums.dart';

part 'category_group.freezed.dart';

@freezed
class CategoryGroup with _$CategoryGroup {
  const factory CategoryGroup({
    required String id,
    required String budgetId,
    required String name,
    required bool hidden,
    required int sortOrder,
    SystemGroupType? systemType,
  }) = _CategoryGroup;

  const CategoryGroup._();

  /// True when this group is managed by the system (not user-renameable/deletable).
  bool get isSystem => systemType != null;
}

// lib/features/categories/domain/category.dart
import 'package:freezed_annotation/freezed_annotation.dart';

part 'category.freezed.dart';

@freezed
class Category with _$Category {
  const factory Category({
    required String id,
    required String groupId,
    required String name,
    required bool hidden,
    String? note,
    required int sortOrder,
    // Non-null only for a credit-card payment category — see docs/03 & docs/04.
    String? linkedAccountId,
  }) = _Category;

  const Category._();

  /// True when this category is the payment category for a credit-card account.
  bool get isCreditCardPayment => linkedAccountId != null;
}

// lib/features/categories/domain/category_repository.dart
abstract interface class CategoryRepository {
  /// Emits all groups in [budgetId] whenever they change.
  Stream<List<CategoryGroup>> watchAllGroups(String budgetId);

  /// Emits all categories in [groupId] whenever they change.
  Stream<List<Category>> watchCategoriesInGroup(String groupId);

  Future<Category?> findCategoryById(String id);

  /// Insert or update by id (upsert semantics).
  Future<void> saveGroup(CategoryGroup group);
  Future<void> saveCategory(Category category);

  Future<void> deleteGroup(String id);
  Future<void> deleteCategory(String id);
}
```

## Implementation notes
- Both entities are `freezed` value objects. The `const CategoryGroup._()` and
  `const Category._()` private constructors are required by `freezed` to allow
  custom getters (`isSystem`, `isCreditCardPayment`) on the class. Run
  `build_runner` after creating the files.
- There are **no money fields** on these entities — assigned/activity/available
  amounts are all derived values computed by the budget engine (M2), never
  stored on the entity itself.
- `CategoryRepository` is a pure `abstract interface class` in `domain`. All
  implementations live in `lib/features/categories/data/` (M1-T17).
- The "Ready to Assign" bucket is **not** a `Category` row — it is a computed
  value (see `docs/04-budgeting-logic.md`). Do not model it here.
- Cross-feature note: `Transaction` and `CategoryBudget` both reference
  `Category` by its `String id`. They import `category.dart` directly from
  this path — that is allowed for domain entities (see `docs/02-architecture.md`).

## Tests to write
File: `test/features/categories/domain/category_group_test.dart`
- Construct a `CategoryGroup` with `systemType: null`; assert `isSystem` is `false`.
- Construct with `systemType: SystemGroupType.creditCardPayments`; assert `isSystem` is `true`.
- `copyWith(name: 'Renamed')` returns a new instance with the new name; original unchanged.
- Two `CategoryGroup` instances with identical fields are `==` and share `hashCode`;
  instances differing in `id` are not equal.

File: `test/features/categories/domain/category_test.dart`
- Construct a `Category` with `linkedAccountId: null`; assert `isCreditCardPayment` is `false`.
- Construct with `linkedAccountId: 'acc-abc'`; assert `isCreditCardPayment` is `true`.
- `copyWith(hidden: true)` on a visible category returns a hidden one; original unchanged.
- Equality: identical fields → equal; differing `id` → not equal.
