## Public API to produce
```dart
// lib/data/database/tables/transactions_table.dart
@DataClassName('TransactionRow')
class Transactions extends Table {
  TextColumn get id => text()();
  TextColumn get accountId => text()();
  TextColumn get date => text()();                       // 'YYYY-MM-DD'
  IntColumn get amount => integer()();                    // signed milliunits
  TextColumn get payeeId => text().nullable()();
  TextColumn get categoryId => text().nullable()();
  TextColumn get memo => text().nullable()();
  TextColumn get cleared => textEnum<ClearedStatus>()();
  BoolColumn get approved => boolean()();
  TextColumn get flagColor => textEnum<FlagColor>().nullable()();
  TextColumn get transferTransactionId =>
      text().nullable().references(Transactions, #id)();
  TextColumn get transferAccountId => text().nullable()();
  TextColumn get scheduledTransactionId => text().nullable()();
  TextColumn get importId => text().nullable()();
  BoolColumn get isSplit => boolean()();
  BoolColumn get deleted => boolean()();
  TextColumn get createdAt => text()();
  TextColumn get updatedAt => text()();

  @override
  Set<Column> get primaryKey => {id};
}

// lib/data/database/tables/sub_transactions_table.dart
@DataClassName('SubTransactionRow')
class SubTransactions extends Table {
  TextColumn get id => text()();
  TextColumn get transactionId => text().references(Transactions, #id)();
  IntColumn get amount => integer()();                    // signed milliunits
  TextColumn get categoryId => text().nullable()();
  TextColumn get payeeId => text().nullable()();
  TextColumn get memo => text().nullable()();
  BoolColumn get deleted => boolean()();

  @override
  Set<Column> get primaryKey => {id};
}
```

## Implementation notes
- **`@DataClassName` is required.** Generated row classes are `TransactionRow`
  and `SubTransactionRow`, avoiding a clash with the domain entities
  `Transaction` / `SubTransaction` (M1-T19).
- `amount` is **signed** milliunits — outflows negative, inflows positive
  (`docs/03`). For a split parent (`isSplit = true`) it equals the sum of its
  sub-rows' `amount`s; the engine, not the schema, enforces that.
- `categoryId` is nullable — null for a split parent, a transfer between two
  on-budget accounts, or an inflow to Ready-to-Assign (`docs/03`).
- Deletes are **soft**: the `deleted` boolean exists so history survives. Never
  hard-delete a transaction or sub-transaction.
- `transferTransactionId` and `sub_transactions.transactionId` reference
  `Transactions` — both tables are in this card, so `.references(Transactions,
  #id)` is included inline above.
- `accountId`, `categoryId`, `payeeId`, `transferAccountId`,
  `scheduledTransactionId` (and `sub_transactions.categoryId` / `payeeId`) are
  logical FKs to tables owned by **other** cards. Leave them as plain
  `text()` / `text().nullable()` columns — M1-T11 adds those FK constraints
  centrally. **NEEDS REVIEW:** M1-T11.
- Timestamps are ISO-8601 text on `transactions`; `sub_transactions` has no
  timestamp columns per the data model — do not add them.
- Do **not** add these tables to `AppDatabase.tables` — that is M1-T11.

## Tests to write
File: `test/data/database/tables/transactions_test.dart`

Define a test-only database in the test file:
```dart
@DriftDatabase(tables: [Transactions, SubTransactions])
class _TestDb extends _$_TestDb {
  _TestDb() : super(NativeDatabase.memory());
  @override
  int get schemaVersion => 1;
}
```

Test cases:
- Insert a `TransactionsCompanion` with a negative `amount`,
  `cleared: ClearedStatus.uncleared`, `flagColor: Value(null)`; select it back
  and verify `amount`, `cleared`, and the null `flagColor`.
- Insert a transaction with `cleared: ClearedStatus.reconciled` and a non-null
  `flagColor`; verify both enum columns round-trip via `textEnum`.
- Insert a transaction with `isSplit: true`, then two `SubTransactionsCompanion`
  rows whose `transactionId` is that transaction; select the sub-rows back and
  verify their `amount`s.
- Insert a transaction with `deleted: true`; verify the flag round-trips.
- Insert a sub-transaction whose `transactionId` does not match any transaction;
  the insert throws (the in-card `.references` FK is enforced).
