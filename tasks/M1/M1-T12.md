## Public API to produce
```dart
// lib/features/accounts/domain/account.dart
import 'package:freezed_annotation/freezed_annotation.dart';
import 'package:budget_app/domain/enums.dart';

part 'account.freezed.dart';

@freezed
class Account with _$Account {
  const factory Account({
    required String id,
    required String budgetId,
    required String name,
    required AccountType type,
    required bool onBudget,
    required bool closed,
    String? note,
    required int sortOrder,
  }) = _Account;
}

// lib/features/accounts/domain/account_repository.dart
abstract interface class AccountRepository {
  /// Emits the full list whenever any account in [budgetId] changes.
  Stream<List<Account>> watchAll(String budgetId);
  Future<Account?> findById(String id);
  /// Insert or update by [Account.id] (upsert semantics).
  Future<void> save(Account account);
  Future<void> delete(String id);
}
```

## Implementation notes
- `Account` is a pure `freezed` value object: immutable, structural equality,
  `copyWith`. Run `build_runner` after creating the file to generate
  `account.freezed.dart`.
- There is **no balance field** on `Account` — the working/cleared/uncleared
  balance is computed by the engine (M2-T02) from the transactions table, not
  stored on the entity.
- `onBudget` mirrors the DB column but is also derivable from `type` via
  `AccountType.isOnBudget`. Both may exist; the stored value is authoritative
  for display; the computed property is a convenience. The two should be
  consistent — the repository implementation (M1-T14) will enforce this.
- `created_at` / `updated_at` are **not** fields on the domain entity; they are
  persistence concerns owned by the data layer.
- `AccountRepository` is an `abstract interface class` in `domain` so controllers
  can depend on it without importing anything from `data/`.

## Tests to write
File: `test/features/accounts/domain/account_test.dart`

Test cases:
- Construct an `Account` with all required fields; verify each field is
  accessible and equals the supplied value.
- `copyWith(name: 'New Name')` returns a new instance with the updated name;
  the original instance is unchanged.
- Two `Account` instances constructed with identical arguments are `==` and
  have the same `hashCode`.
- Two instances differing only in `id` are not equal.
- `AccountType.isOnBudget` is `true` for `checking` and `savings`; `false` for
  `asset` and `liability`. (Guards against accidental enum reordering.)
