## Public API to produce
```dart
// lib/data/daos/transactions_dao.dart
@DriftAccessor(tables: [Transactions, SubTransactions, Accounts])
class TransactionsDao extends DatabaseAccessor<AppDatabase>
    with _$TransactionsDaoMixin {
  TransactionsDao(super.db);

  /// Non-deleted transactions for [accountId], `date` descending.
  Stream<List<TransactionRow>> watchByAccount(String accountId);

  Future<TransactionRow?> findById(String id);

  /// Non-deleted sub-transactions of [transactionId].
  Future<List<SubTransactionRow>> subTransactionsOf(String transactionId);

  /// Non-deleted transactions for every account in [budgetId]
  /// (joins `transactions` → `accounts`).
  Future<List<TransactionRow>> allForBudget(String budgetId);

  Future<void> upsertTransaction(TransactionsCompanion transaction);

  /// Atomically replace the sub-transactions of [parentId] with [subs].
  Future<void> replaceSubTransactions(
    String parentId,
    List<SubTransactionsCompanion> subs,
  );

  /// Soft delete — sets `deleted = true` on the transaction.
  Future<void> softDelete(String id);
}
```

## Implementation notes
- `Accounts` is in the `@DriftAccessor` table list only so `allForBudget` can
  join `transactions.accountId` to `accounts.id` and filter by
  `accounts.budgetId`. Run `build_runner` for `_$TransactionsDaoMixin`.
- Every read query filters `deleted.equals(false)` — soft-deleted rows are never
  returned (`docs/03`).
- `watchByAccount` orders by `date` descending, then ends in `.watch()`.
- `replaceSubTransactions` runs inside `transaction(() async { ... })`: first
  delete existing sub rows for `parentId`, then insert the new companions. It is
  the single write path for a split's children, keeping parent and children
  consistent.
- `softDelete` updates the `deleted` column to `true` and `updatedAt` to the
  caller-supplied value — accept `updatedAt` is set by the repository
  (M1-T21); here just write `TransactionsCompanion(deleted: Value(true), ...)`
  via an `update` statement on the matching `id`.
- The DAO returns Drift row types only; mapping is the repository's job
  (M1-T21).

## Tests to write
File: `test/data/daos/transactions_dao_test.dart`

Real in-memory database, closed in `tearDown`. `PRAGMA foreign_keys = ON` is
active (M1-T11): seed `budgets` → `accounts` parents first.

Test cases:
- `upsertTransaction` a row, then `findById` returns it.
- `watchByAccount` emits non-deleted transactions newest-`date` first and omits
  another account's rows.
- `softDelete` a transaction; `watchByAccount` no longer emits it but
  `findById` still finds the row with `deleted = true`.
- `replaceSubTransactions` inserts a split's children; `subTransactionsOf`
  returns them; calling it again with a different list replaces them (the old
  children are gone, only the new set remains).
- `allForBudget` returns transactions from two accounts of the same budget and
  excludes a transaction from an account in another budget.
