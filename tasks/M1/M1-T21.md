## Implementation notes
- **Date mapping.** The entity's `date` is a `DateTime`; the column is
  `YYYY-MM-DD` text. Map with a date-only ISO string —
  `'${d.year}-${two(d.month)}-${two(d.day)}'` out, `DateTime.parse` in. Do not
  store a time component. Keep this in the mapper file.
- **Money mapping.** `amount` int milliunits ↔ `Money`.
- **Timestamps.** `Transaction` carries no timestamps; `save` preserves
  `createdAt` exactly as M1-T14: read the existing row, reuse its `createdAt`,
  set `updatedAt = now`. `sub_transactions` has no timestamp columns, so
  `subTransactionToCompanion` takes none.
- **Split save.** When `transaction.isSplit`, `save` upserts the parent then
  calls `dao.replaceSubTransactions(transaction.id, subTransactions.map(...))`.
  When it is not a split, `subTransactions` is ignored. Both steps for a split
  must be atomic — wrap them in a `db.transaction(...)` (or rely on the DAO's
  atomic `replaceSubTransactions` plus a preceding parent upsert inside one
  `transaction`).
- `allForBudget` maps `dao.allForBudget` rows to entities; this is the budget
  engine's feed (M2).
- The impl lives in `data/`; `package:drift` and the database import are
  allowed here. The domain layer stays Drift-free.
- No domain exceptions in M1 — `findById` returns `null` for not-found.

## Tests to write
File: `test/features/transactions/data/transaction_mappers_test.dart`
- `transactionFromRow` / `transactionToCompanion` round-trip every field,
  including the date (a `DateTime` ↔ `YYYY-MM-DD`), a negative `Money` amount,
  null `categoryId`, and a non-null `flagColor`.
- `subTransactionFromRow` / `subTransactionToCompanion` round-trip every field.

File: `test/features/transactions/data/transaction_repository_impl_test.dart`
Real in-memory database, closed in `tearDown`; seed `budgets` → `accounts`
parents first.
- `save` a simple transaction, then `findById` returns the domain entity with
  `date` and `amount` intact.
- `save` a split transaction with two sub-transactions; `subTransactionsOf`
  returns both; re-saving with one sub replaces the set.
- `save` twice with the same `id` updates the row and preserves `createdAt`.
- `softDelete` removes the transaction from `watchByAccount` but `findById`
  still returns it with `deleted = true`.
- `allForBudget` returns transactions across the budget's accounts.
