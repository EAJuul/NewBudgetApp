## Public API to produce
```dart
enum AccountType {
  checking, savings, cash, creditCard, lineOfCredit, asset, liability;

  /// On-budget for the first five; tracking accounts (asset/liability) are off.
  bool get isOnBudget;
  bool get isCreditCard;
}

enum ClearedStatus { uncleared, cleared, reconciled }

enum FlagColor { red, orange, yellow, green, blue, purple }

enum ScheduleFrequency {
  daily, weekly, everyOtherWeek, twiceAMonth, every4Weeks,
  monthly, everyOtherMonth, every3Months, every6Months, yearly,
}

enum TargetType {
  monthlyFunding, targetBalance, targetBalanceByDate, monthlySpending,
}

enum SystemGroupType { creditCardPayments, internal }
```

## Implementation notes
- Drift will persist these with `.textEnum<T>()`, storing the enum **name**.
  Do not reorder or rename members after M1-T11 ships without a schema
  migration — the name is the stored value.
- No manual `TypeConverter` is needed for enums.

## Tests to write
File: `test/domain/enums_test.dart`
- `AccountType.isOnBudget` is true for `checking`..`lineOfCredit`, false for
  `asset` and `liability`.
- `AccountType.isCreditCard` is true only for `creditCard`.
- The `.name` of each member equals the expected string (guards the stored
  value against accidental renames).
