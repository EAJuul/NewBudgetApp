## Public API to produce
```dart
// lib/features/accounts/data/account_mappers.dart
Account accountFromRow(AccountRow row);

/// Build a companion for an upsert. The caller owns timestamp values so that
/// `createdAt` is preserved across updates (see implementation notes).
AccountsCompanion accountToCompanion(
  Account account, {
  required String createdAt,
  required String updatedAt,
});

// lib/features/accounts/data/account_repository_impl.dart
class AccountRepositoryImpl implements AccountRepository {
  AccountRepositoryImpl(AppDatabase db);   // constructs its own AccountsDao(db)

  @override
  Stream<List<Account>> watchAll(String budgetId);
  @override
  Future<Account?> findById(String id);
  @override
  Future<void> save(Account account);
  @override
  Future<void> delete(String id);
}
```

## Implementation notes
- `accountFromRow` maps every domain field; the row's `createdAt` / `updatedAt`
  are **not** carried onto the entity — they are persistence-only (`Account`
  has no such fields, M1-T12).
- `watchAll` maps the DAO's `Stream<List<AccountRow>>` with
  `.map((rows) => rows.map(accountFromRow).toList())`.
- **Timestamps in `save`.** `Account` carries no timestamps, but the table
  columns are non-null. `save` must:
  1. `final existing = await dao.findById(account.id);`
  2. `final now = DateTime.now().toUtc().toIso8601String();`
  3. `createdAt = existing?.createdAt ?? now;` `updatedAt = now;`
  4. `await dao.upsert(accountToCompanion(account, createdAt: ..., updatedAt: ...));`
  This preserves the original `createdAt` on an update.
- `save` has upsert semantics — it is the single write path for both create and
  update, matching the `AccountRepository` contract (M1-T12).
- `AccountRepositoryImpl` lives in `data/`, so importing `package:drift` and the
  database here is allowed; the domain `Account` / `AccountRepository` must stay
  Drift-free.
- M1 repository methods do not throw domain exceptions: `findById` returns
  `null` for not-found and underlying DB failures propagate. Typed `core/error/`
  exceptions are introduced later when a controller needs them.

## Tests to write
File: `test/features/accounts/data/account_mappers_test.dart`
- `accountFromRow` produces an `Account` whose every field equals the row's.
- `accountToCompanion` produces a companion whose `Value`s match the entity and
  the supplied `createdAt` / `updatedAt`.
- A nullable field (`note`) round-trips as null through both mappers.

File: `test/features/accounts/data/account_repository_impl_test.dart`
Use a real in-memory database (`AppDatabase.forTesting(NativeDatabase.memory())`,
closed in `tearDown`); seed a parent `budgets` row first.
- `save` then `findById` returns the saved account as a domain entity.
- `save` twice with the same `id` updates the row and keeps the original
  `createdAt` (read the row back through the DAO to confirm).
- `watchAll` emits the budget's accounts as `Account` entities.
- `delete` removes the account; `findById` returns `null`.
