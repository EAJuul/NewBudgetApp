## Public API to produce
```dart
// lib/features/targets/domain/target.dart
@freezed
class Target with _$Target {
  const factory Target({
    required String id,
    required String categoryId,
    required TargetType type,
    required Money amount,
    MonthKey? targetMonth,            // set only for targetBalanceByDate
  }) = _Target;
}

// lib/features/targets/domain/target_repository.dart
abstract interface class TargetRepository {
  /// Every target (MVP is single-budget), re-emitted on change.
  Stream<List<Target>> watchAll();
  /// The 0..1 target for [categoryId].
  Future<Target?> findForCategory(String categoryId);
  /// Insert or update — at most one target per category (UNIQUE).
  Future<void> save(Target target);
  Future<void> deleteForCategory(String categoryId);
}

// lib/data/daos/targets_dao.dart
@DriftAccessor(tables: [Targets])
class TargetsDao extends DatabaseAccessor<AppDatabase> with _$TargetsDaoMixin {
  TargetsDao(super.db);
  Stream<List<TargetRow>> watchAll();
  Future<TargetRow?> findByCategory(String categoryId);
  Future<void> upsert(TargetsCompanion target);
  Future<void> deleteByCategory(String categoryId);
}

// lib/features/targets/data/target_mappers.dart
Target targetFromRow(TargetRow row);
TargetsCompanion targetToCompanion(
  Target target, {
  required String createdAt,
  required String updatedAt,
});

// lib/features/targets/data/target_repository_impl.dart
class TargetRepositoryImpl implements TargetRepository {
  TargetRepositoryImpl(AppDatabase db);   // constructs its own TargetsDao(db)
}
```

## Implementation notes
- The entity stores `amount` as `Money` and `targetMonth` as `MonthKey?`. The
  mapper converts: `amount` int ↔ `Money`; `targetMonth` text ↔
  `MonthKey.parse` / `.toIso()`, preserving null.
- `targetMonth` is non-null **only** for `TargetType.targetBalanceByDate`; do
  not validate the combination in the data layer — that is a domain/UI concern
  (M6).
- `targets` has `created_at` / `updated_at`; `save` preserves `createdAt` across
  updates (same pattern as M1-T14).
- The `UNIQUE(category_id)` constraint (M1-T07) gives upsert-by-category: the
  DAO's `upsert` plus `insertOnConflictUpdate`. `save` looks up the existing row
  by `categoryId` to reuse its `id` and `createdAt`.
- `findForCategory` / `deleteForCategory` key on `categoryId` because that is
  the natural identity of "the category's target."
- The impl lives in `data/`; the domain entity and interface stay Drift-free.
