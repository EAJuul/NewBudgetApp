## Goal
The single place money crosses the domain ↔ display boundary: format a `Money`
into a human-readable string, and parse user-typed text back into `Money`.
No other code formats or parses money.

## Public API to produce
```dart
// lib/core/money/currency_formatter.dart
/// Converts [Money] to/from display text. The one money display boundary.
class CurrencyFormatter {
  const CurrencyFormatter._();

  /// Formats [amount] for display.
  /// `Money(1234560)` → `"$1,234.56"`; `Money(-50000)` → `"-$50.00"`.
  /// [symbol] is the currency symbol; [decimalDigits] the fraction digits.
  static String format(
    Money amount, {
    String symbol = r'$',
    int decimalDigits = 2,
  });

  /// Parses user input into [Money]; returns null when [input] is not a valid
  /// number. Accepts an optional leading sign, the currency [symbol], grouping
  /// commas, and surrounding whitespace — e.g. `"1,234.56"`, `"-50"`, `"$12"`.
  /// An empty or blank string returns null.
  static Money? tryParse(
    String input, {
    String symbol = r'$',
    int decimalDigits = 2,
  });
}
```

## Implementation notes
- **Milestone scope.** MVP is single-currency; `symbol` and `decimalDigits`
  default to a US-dollar style. They are parameters (not hard-coded) so a later
  task can feed `Budget.currencyCode` / `Budget.currencyDecimalDigits` in.
  Do **not** wire the `Budget` entity in here — keep this file pure and
  budget-agnostic.
- **`format`:**
  - `Money` is integer milliunits (1 unit = 1000 milliunits). Convert to a
    display value with `decimalDigits` fraction digits, rounding **half-up**:
    e.g. `decimalDigits == 2` → divide milliunits by 10 and round, giving
    hundredths. `Money(1234560)` → `1234.56`.
  - Group the integer part in threes with `,`.
  - Negative amounts: a single leading `-` **before** the symbol —
    `"-$50.00"`. Zero formats without a sign — `"$0.00"`.
  - Result shape: `<optional ->` + `symbol` + `<grouped integer>` + `.` +
    `<zero-padded fraction>` (omit the `.` and fraction when
    `decimalDigits == 0`).
- **`tryParse`:**
  - Trim; return `null` for an empty/blank string.
  - Strip the `symbol`, grouping commas, and internal spaces; allow one
    optional leading `+`/`-`.
  - Parse the remaining `<int>` or `<int>.<frac>` into milliunits: scale by
    `1000` and add the fraction scaled to milliunits; reject more fraction
    digits than `decimalDigits` allows, any other non-digit character, or an
    empty numeric body → `null`.
  - Apply the sign; build the result with the `Money` constructor.
  - `format` and `tryParse` round-trip for values expressible in
    `decimalDigits` (e.g. `tryParse(format(m))` recovers `m`).
- No `double` anywhere — milliunits and display digits are all integer
  arithmetic (`docs/09`). Parse the fraction as an integer string, not via
  `double.parse`.

## Tests to write
File: `test/core/money/currency_formatter_test.dart`
- `format` of zero → `"$0.00"`.
- `format` of a positive whole amount, a sub-unit amount, and a value needing
  thousands grouping → exact expected strings.
- `format` of a negative amount → leading `-` before the `$`.
- `format` with `decimalDigits: 0` omits the fraction; with a custom `symbol`
  uses it.
- `format` rounds half-up when milliunits do not divide evenly into the
  requested `decimalDigits`.
- `tryParse` of `""` / blank / non-numeric text → `null`.
- `tryParse` of `"1,234.56"`, `"-50"`, `"$12"`, `" 7.5 "` → exact `Money`
  milliunit values.
- `tryParse` rejects too many fraction digits for `decimalDigits`.
- Round-trip: `tryParse(format(m))` equals `m` for a sample of positive,
  negative, and zero amounts.
