## Public API to produce
```dart
// lib/features/transactions/domain/transaction.dart
@freezed
class Transaction with _$Transaction {
  const factory Transaction({
    required String id,
    required String accountId,
    required DateTime date,
    required Money amount,             // signed milliunits
    String? payeeId,
    String? categoryId,                // null: split, transfer, or inflow-to-RTA
    String? memo,
    required ClearedStatus cleared,
    required bool approved,
    FlagColor? flagColor,
    String? transferTransactionId,
    String? transferAccountId,
    String? scheduledTransactionId,
    String? importId,
    required bool isSplit,
    required bool deleted,
  }) = _Transaction;

  const Transaction._();

  /// True when this row is one side of a transfer pair.
  bool get isTransfer => transferTransactionId != null;
}

// lib/features/transactions/domain/sub_transaction.dart
@freezed
class SubTransaction with _$SubTransaction {
  const factory SubTransaction({
    required String id,
    required String transactionId,
    required Money amount,             // signed milliunits
    String? categoryId,
    String? payeeId,
    String? memo,
    required bool deleted,
  }) = _SubTransaction;
}

// lib/features/transactions/domain/transaction_repository.dart
abstract interface class TransactionRepository {
  /// Non-deleted transactions for [accountId], most recent `date` first.
  Stream<List<Transaction>> watchByAccount(String accountId);

  Future<Transaction?> findById(String id);

  /// The non-deleted sub-transactions of a split parent.
  Future<List<SubTransaction>> subTransactionsOf(String transactionId);

  /// Every non-deleted transaction across all accounts in [budgetId] — the
  /// budget engine's input (M2).
  Future<List<Transaction>> allForBudget(String budgetId);

  /// Insert or update [transaction]; when it is a split, replace its
  /// sub-transactions with [subTransactions] atomically.
  Future<void> save(
    Transaction transaction, {
    List<SubTransaction> subTransactions,
  });

  /// Soft delete — sets `deleted = true`, preserving history.
  Future<void> softDelete(String id);
}
```

## Implementation notes
- Both entities are `freezed` value objects. `Transaction` needs the private
  `const Transaction._();` constructor so the `isTransfer` getter is allowed.
  `SubTransaction` needs no getter, so it needs no private constructor. Run
  `build_runner` after creating the files.
- `date` is a `DateTime` on the entity; the data layer (M1-T21) maps it to/from
  the `YYYY-MM-DD` text column. The entity carries **no** `createdAt` /
  `updatedAt` — those are persistence-only.
- `amount` is **signed** `Money`: outflows negative, inflows positive.
- A split parent has `isSplit = true`, `categoryId = null`, and its categorised
  amounts live in `SubTransaction` rows whose `amount`s sum to the parent's
  (the engine enforces this, M2).
- `save`'s `subTransactions` parameter defaults to an empty list; it is ignored
  unless `transaction.isSplit` is true.
- `TransactionRepository` is a pure `abstract interface class` in `domain`.
