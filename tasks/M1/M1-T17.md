## Implementation notes
- **Timestamps differ by table.** `category_groups` has no `created_at` /
  `updated_at` columns (`docs/03`), so `categoryGroupToCompanion` takes no
  timestamp arguments. `categories` does have them, so `saveCategory` follows
  the same preserve-`createdAt` pattern as M1-T14: read the existing row,
  `createdAt = existing?.createdAt ?? now`, `updatedAt = now`.
- `watch*` methods map the DAO streams with `rows.map(...FromRow).toList()`.
- `saveGroup` / `saveCategory` have upsert semantics — one write path each.
- `categoryFromRow` sets `linkedAccountId` from the row; the entity getter
  `isCreditCardPayment` (M1-T15) then derives from it — do not recompute it in
  the mapper.
- The impl lives in `data/`, so `package:drift` and the database import are
  allowed; the domain entities and interface stay Drift-free.
- No domain exceptions in M1 — `findCategoryById` returns `null` for not-found.
