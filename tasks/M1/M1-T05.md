## Public API to produce
```dart
// lib/data/database/tables/budgets_table.dart
@DataClassName('BudgetRow')
class Budgets extends Table {
  TextColumn get id => text()();
  TextColumn get name => text()();
  TextColumn get currencyCode => text()();
  IntColumn get currencyDecimalDigits => integer()();
  TextColumn get dateFormat => text()();
  TextColumn get createdAt => text()();
  TextColumn get updatedAt => text()();

  @override
  Set<Column> get primaryKey => {id};
}

// lib/data/database/tables/accounts_table.dart
@DataClassName('AccountRow')
class Accounts extends Table {
  TextColumn get id => text()();
  TextColumn get budgetId => text()();
  TextColumn get name => text()();
  TextColumn get type => textEnum<AccountType>()();
  BoolColumn get onBudget => boolean()();
  BoolColumn get closed => boolean()();
  TextColumn get note => text().nullable()();
  IntColumn get sortOrder => integer()();
  TextColumn get createdAt => text()();
  TextColumn get updatedAt => text()();

  @override
  Set<Column> get primaryKey => {id};
}
```

## Implementation notes
- Each table is annotated `@DataClassName('<Entity>Row')` so Drift's generated
  row class (`BudgetRow`, `AccountRow`) does not collide with the domain entity
  of the same name (`Budget` — M1-T25; `Account` — M1-T12). Every table in this
  schema follows the `<Entity>Row` convention.
- Primary keys are UUIDs generated in Dart, stored as `TEXT` — use `text()()`
  with no `autoIncrement`.
- `AccountType` is persisted by its `.name` via `textEnum<AccountType>()`. The
  enum is defined in `lib/domain/enums.dart` (M1-T03). Do not reorder or rename
  enum members after M1-T11 ships without a schema migration.
- Timestamps `createdAt` / `updatedAt` are ISO-8601 text strings; use
  `TextColumn`, not `DateTimeColumn`. See `docs/03-data-model.md` — Dates.
- `budgetId` in `Accounts` is a logical FK to `budgets.id`. Adding
  `.references(Budgets, #id)` enforces the constraint in SQLite; include it
  since both tables are defined in this task.
- Do **not** add these tables to `AppDatabase.tables` yet — that is M1-T11.

## Tests to write
File: `test/data/database/tables/budgets_accounts_test.dart`

Define a test-only database in the test file and open it in-memory:
```dart
@DriftDatabase(tables: [Budgets, Accounts])
class _TestDb extends _$_TestDb {
  _TestDb() : super(NativeDatabase.memory());
  @override
  int get schemaVersion => 1;
}
```

Test cases:
- Insert a `BudgetsCompanion` row and select it back; verify `currencyCode` and
  `currencyDecimalDigits` match.
- Insert `AccountsCompanion` rows for `AccountType.checking`, `AccountType.creditCard`,
  and `AccountType.asset`; verify `type` round-trips correctly via `textEnum`.
- Insert an account with `note: Value(null)`; read it back and assert `note` is
  null.
- Insert an account with `closed: Value(false)` and `onBudget: Value(true)`;
  verify both boolean values survive the round-trip.
