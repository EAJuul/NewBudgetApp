## Implementation notes
- **MVP single budget.** The `budgets` table holds exactly one row after
  onboarding. `getPrimary` / `watchPrimary` return that row (or null before it
  exists). `save` upserts it.
- `budgets` has `created_at` / `updated_at`; `save` preserves `createdAt` across
  updates (same pattern as M1-T14).
- The `Budget` entity carries config fields only — not timestamps.
- **`SettingsStore`** wraps the `settings` key-value table (PK is `key`, no UUID
  — `docs/08` / M1-T08). `set` upserts via `insertOnConflictUpdate`; `get`
  returns the `value` or null; `watch(key)` emits the value and re-emits on
  change. `settings` has no timestamps.
- Both impls live in `data/`; the domain entity and interfaces stay Drift-free.
