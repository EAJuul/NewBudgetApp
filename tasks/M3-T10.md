## Implementation notes
- `Scaffold` + `AppBar`. The app bar title is the account name (available once
  the controller resolves; show a neutral title meanwhile). Add an edit action
  in the app bar → `context.go('/accounts/$accountId/edit')`.
- Watch `accountDetailControllerProvider(accountId)` and switch with `.when`:
  - `loading` → centered `CircularProgressIndicator`.
  - `error` → centered error message (e.g. "Account not found").
  - `data` → a balance header card showing the prominent **working** balance
    and, beneath it, the **cleared** and **uncleared** balances, each labelled
    and formatted via `CurrencyFormatter`.
- Below the header, render a short placeholder section for transactions (a
  simple labelled `Text`, e.g. "Transactions") — the transaction list itself
  arrives in milestone M4 and is out of scope here.
- The widget is dumb: all values come from `AccountDetail`; no computation in
  the screen.
