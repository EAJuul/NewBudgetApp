## Test harness
- Wrap each pumped widget in a `ProviderScope` whose `overrides` replace
  `appDatabaseProvider` with `AppDatabase.forTesting(NativeDatabase.memory())`.
- Seed test data through the repository providers (a `Budget` first, then
  accounts / transactions) before pumping; `addTearDown` to dispose.
- Pump screens that use `context.go` inside a `MaterialApp.router` bound to
  `routerProvider`; pump leaf widgets inside a plain `MaterialApp` with a
  `Scaffold`.
- `await tester.pumpAndSettle()` to let streams and `build_runner` providers
  resolve.

## Tests to write
**`AccountListScreen`** — loading shows a progress indicator; an empty budget
shows the empty state; with accounts seeded, one `AccountListItem` per account
renders; the add affordance is present.

**`AccountListItem`** — shows the account name; shows the formatted working
balance once it resolves; a closed account shows the "Closed" marker; tapping
invokes `onTap`.

**`CreateAccountScreen`** — submitting an empty name shows the validator
error; a valid submit persists an account (assert via the repository); the
submit control is disabled while the controller is loading.

**`EditAccountScreen`** — pre-fills the loaded account's name / type / note;
saving updates the account (assert via the repository); a not-found id shows
the error state.

**`AccountDetailScreen`** — loading shows a progress indicator; the data state
shows working / cleared / uncleared balances; an unknown id shows the error
state.

**`AccountGroups`** — accounts split into Budget / Tracking / Closed sections;
an empty section renders no header; tapping a row invokes `onAccountTap`.

**`NetWorthSummary`** — shows the summed working balance of non-closed
accounts; a closed account is excluded from the total.

**Navigation flow** — from `/accounts`, tapping a row navigates to
`/accounts/:id`; the detail screen's edit action navigates to
`/accounts/:id/edit`; the add affordance navigates to `/accounts/new`.
