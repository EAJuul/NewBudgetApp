## Public API to produce
```dart
// lib/domain/budgeting/ready_to_assign_calculator.dart
/// Ready to Assign = total inflow to RTA − total assigned.
///
/// Positive: money waiting for a job. Negative: the user has assigned more than
/// they have.
Money computeReadyToAssign({
  required Iterable<Account> accounts,
  required Iterable<Transaction> transactions,
  required Iterable<CategoryBudget> categoryBudgets,
});
```

## Implementation notes
- Build a `Set<String>` of on-budget account ids: `accounts` where
  `account.onBudget == true`.
- **Inflow to RTA — engine interpretation.** `docs/04` describes RTA inflow as
  transactions in the special "Inflow: Ready to Assign" bucket, but `docs/03`
  defines no such category row. This card resolves the ambiguity: a transaction
  counts toward `totalInflowToRTA` when **all** hold —
  - its `accountId` is in the on-budget set;
  - `deleted == false`;
  - `isSplit == false`;
  - `transferTransactionId == null` (it is not a transfer);
  - `categoryId == null`.
  This captures on-budget starting-balance transactions and income left
  uncategorised to RTA. `totalInflowToRTA` is the sum of those `amount`s.
  **NEEDS REVIEW:** confirm this rule against the intended RTA-inflow modelling
  before M2-T16.
- `totalAssigned` = the sum of `assigned` across **all** `categoryBudgets`
  rows passed in (every category, every month).
- `RTA = totalInflowToRTA − totalAssigned`. Sum with `Money` arithmetic from
  `Money.zero()`.
- `assigned` may be negative — it still flows through the sum unchanged.

## Tests to write
File: `test/domain/budgeting/ready_to_assign_calculator_test.dart`
Build data with `BudgetFixture` (M2-T01); use hand-computed milliunit totals.
- A single on-budget account with a starting-balance inflow and no assignments →
  RTA equals that inflow.
- Assigning money reduces RTA by exactly the assigned total.
- Assigning more than the inflow yields a **negative** RTA.
- An inflow on an **off-budget** (tracking) account does **not** affect RTA.
- A `deleted` inflow is excluded.
- A categorised transaction (non-null `categoryId`) is **not** counted as RTA
  inflow.
- A transfer (`transferTransactionId` set) is **not** counted as RTA inflow.
