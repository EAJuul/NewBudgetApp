## Public API to produce
```dart
// lib/domain/budgeting/target_progress_calculator.dart
@freezed
class TargetProgress with _$TargetProgress {
  const factory TargetProgress({
    required Money needed,    // still required this month; Money.zero() when met
    required bool isMet,
  }) = _TargetProgress;
}

/// Progress of [target] for [month], given the category's [assigned] and
/// [available] amounts for that month (from M2-T03 / M2-T04 / the budget line).
TargetProgress computeTargetProgress({
  required Target target,
  required MonthKey month,
  required Money assigned,
  required Money available,
});
```

## Implementation notes
Per `docs/04` — Target progress. `T` is `target.amount`.
- **`monthlyFunding`** — compare against `assigned`:
  `isMet = assigned >= T`; `needed = max(Money.zero(), T − assigned)`.
- **`monthlySpending`** — compare against `available` (prior rollover counts):
  `isMet = available >= T`; `needed = max(Money.zero(), T − available)`.
- **`targetBalance`** — compare against `available`:
  `isMet = available >= T`; `needed = max(Money.zero(), T − available)`.
- **`targetBalanceByDate`** — spread the remainder over the months left:
  - `monthsLeft = month.monthsUntil(target.targetMonth!)`, clamped to a minimum
    of `1`.
  - `remaining = T − available`.
  - `isMet = remaining <= Money.zero()`.
  - `needed = isMet ? Money.zero() : Money(ceilDiv(remaining.milliunits,
    monthsLeft))`, where `ceilDiv(a, b)` for `a > 0, b > 0` is `(a + b - 1) ~/ b`.
- `target.targetMonth` is non-null only for `targetBalanceByDate`. For that
  type, treat a null `targetMonth` as a malformed target — assume callers do not
  pass one (the data layer permits it, but the UI, M6, guarantees it). A
  defensive `ArgumentError` is acceptable but not required.
- Use a `switch` over `TargetType` with no `default`, so a future enum member is
  a compile error. `Money` arithmetic only — no `double`.
- `TargetProgress` is `freezed`; run `build_runner` for the generated part.

## Tests to write
File: `test/domain/budgeting/target_progress_calculator_test.dart`
Hand-computed milliunit values; cover all four types.
- `monthlyFunding`: under-assigned → `needed` is the gap, `isMet` false;
  fully assigned → `needed` zero, `isMet` true; over-assigned → `isMet` true.
- `monthlySpending`: a category with rollover `available` ≥ `T` is met even
  when this month's `assigned` is zero.
- `targetBalance`: `available` below / equal / above `T`.
- `targetBalanceByDate`: with N months left, `needed` is the remainder divided
  across N, rounded **up** (verify the ceil with a non-divisible remainder);
  `monthsLeft` clamps to 1 when `month` is at or past `targetMonth`; an already
  met balance gives `needed` zero.
