## Routes to add
Add these `GoRoute`s to the existing `routes` list:

| Path | Screen | Notes |
|---|---|---|
| `/accounts` | `AccountListScreen` | the flat list |
| `/accounts/new` | `CreateAccountScreen` | declare **before** `/accounts/:id` |
| `/accounts/:id` | `AccountDetailScreen` | `accountId` from `state.pathParameters['id']!` |
| `/accounts/:id/edit` | `EditAccountScreen` | `accountId` from `state.pathParameters['id']!` |

## Implementation notes
- **Ordering matters.** `go_router` matches routes in declaration order, and
  the pattern `/accounts/:id` would also match the literal `/accounts/new`.
  Declare `/accounts/new` **before** `/accounts/:id` so the literal wins.
- For the two parameterised routes, read the id with
  `state.pathParameters['id']!` and pass it as the `accountId` constructor
  argument. This `!` is justified — `go_router` guarantees the parameter is
  present for a matched `:id` route (the one allowed-`!` exception in
  `docs/09`); add a one-line comment saying so.
- Keep each `builder` a one-liner returning the screen. Do not add redirects,
  guards, shell routes, or transitions in this task.
- Do not change how `routerProvider` itself is declared.
