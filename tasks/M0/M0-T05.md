- **Milestone:** M0
- **Dependencies:** M0-T03, M0-T04

## Goal
Replace the placeholder `home:` with a `go_router` configuration and a shared
Material 3 theme so feature milestones can register their routes.

## Context — read first
- `docs/02-architecture.md`
- `docs/09-coding-standards.md` — State management

## Files
Create:
- `lib/app/router.dart` — the `GoRouter`, exposed as a Riverpod provider, with
  one placeholder route `/` rendering a `HomePlaceholderScreen`.
- `lib/app/theme.dart` — `lightTheme` and `darkTheme`.
Edit:
- `lib/app/app.dart` — convert `BudgetApp` to a `ConsumerWidget` using
  `MaterialApp.router`.
- `test/app_test.dart` — update the smoke test for the router.

## Public API to produce
```dart
// router.dart
@riverpod
GoRouter router(Ref ref);   // use the ref type your riverpod_generator
                            // version expects (Ref in 2.6+)

// theme.dart
ThemeData get lightTheme;   // Material 3, seed colour 0xFF2E7D32
ThemeData get darkTheme;
```

## Implementation notes
- Keep exactly one placeholder route; feature milestones add the rest.
- `app.dart` reads the router provider with `ref.watch` and passes
  `lightTheme` / `darkTheme` plus `themeMode: ThemeMode.system`.
- Run `dart run build_runner build` to generate the router provider.
