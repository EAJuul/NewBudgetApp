# M0-T01 — Initialize the Flutter project

- **Milestone:** M0
- **Dependencies:** none
- **Estimated new code:** S (mostly generated by `flutter create`)

## Goal
Generate the native iOS/Android platform folders and confirm the toolchain
satisfies the project's dependency requirements.

## Context — read first
- `docs/08-ci-cd.md` — Reproducible toolchain
- `docs/02-architecture.md`

## Steps
1. Verify the toolchain:
   ```
   flutter --version
   ```
   Confirm Flutter ≥ 3.24 and Dart ≥ 3.5. The SDK at
   `/Users/emilhansen/code/flutter` reports `3.7.0`; if `flutter --version`
   confirms a version below the minimum, run `flutter upgrade` or install a
   current stable SDK before continuing.
2. Generate platform scaffolding in a temp directory so the existing
   `pubspec.yaml`, `lib/`, `test/`, and `analysis_options.yaml` are not
   overwritten:
   ```
   flutter create --project-name budget_app --org com.example --platforms=android,ios /tmp/budget_app_tmp
   ```
3. Copy the native folders into the repo root:
   ```
   cp -r /tmp/budget_app_tmp/android /tmp/budget_app_tmp/ios .
   cp /tmp/budget_app_tmp/.metadata .
   ```
4. Run `flutter pub get`, then `flutter run` on a simulator and confirm the
   scaffold app ("NewBudgetApp — scaffold ready") launches.

## Files
Generated: `android/`, `ios/`, `.metadata`. Do **not** modify `lib/`,
`pubspec.yaml`, `test/`, or `analysis_options.yaml`.
