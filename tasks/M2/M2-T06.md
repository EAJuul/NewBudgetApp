## Goal
Define the immutable `MonthBudget` aggregate — the complete computed state of
one budget month: Ready-to-Assign plus one line per category. This is what
`BudgetService` returns (M2-T07) and the budget screen renders (M5).

## Public API to produce
```dart
// lib/domain/budgeting/month_budget.dart
@freezed
class CategoryBudgetLine with _$CategoryBudgetLine {
  const factory CategoryBudgetLine({
    required String categoryId,
    required Money assigned,
    required Money activity,
    required Money available,
  }) = _CategoryBudgetLine;
}

@freezed
class MonthBudget with _$MonthBudget {
  const factory MonthBudget({
    required MonthKey month,
    required Money readyToAssign,
    required List<CategoryBudgetLine> lines,
  }) = _MonthBudget;

  const MonthBudget._();

  /// The line for [categoryId], or null when the category has no line.
  CategoryBudgetLine? lineFor(String categoryId);
}
```

## Implementation notes
- Both classes are pure `freezed` value objects. `MonthBudget` needs the private
  `const MonthBudget._();` constructor so the `lineFor` getter is allowed. Run
  `build_runner` for the `.freezed.dart` parts.
- `lineFor` does a linear search of `lines`; use `firstWhereOrNull` from
  `package:collection` (a declared dependency) — do not throw when absent.
- `MonthBudget` is a **pure data holder** — it performs no calculation. The
  values are produced by the calculators (M2-T03..T05) and assembled by
  `BudgetService` (M2-T07).
- A `CategoryBudgetLine` exists for every category in the budget, even one with
  zero assigned / activity / available.
