## Public API to produce
```dart
// lib/domain/budgeting/category_activity_calculator.dart
/// Activity for [categoryId] within [month]. Normally negative (spending).
///
/// [transactions] and [subTransactions] may span any categories/months — the
/// function filters internally. Sub-transactions are dated by their **parent**
/// transaction's `date`.
Money computeCategoryActivity({
  required String categoryId,
  required MonthKey month,
  required Iterable<Transaction> transactions,
  required Iterable<SubTransaction> subTransactions,
});
```

## Implementation notes
- Build a `Map<String, Transaction>` keyed by `Transaction.id` so each
  sub-transaction can find its parent (for the parent's `date` and `deleted`).
- **Direct contribution.** Add `transaction.amount` when:
  `transaction.categoryId == categoryId` &&
  `transaction.isSplit == false` &&
  `transaction.deleted == false` &&
  `month.contains(transaction.date)`.
- **Split contribution.** Add `sub.amount` when:
  `sub.categoryId == categoryId` &&
  `sub.deleted == false` &&
  the parent exists, is not `deleted`, and `month.contains(parent.date)`.
- A split parent's own `amount` is never counted — its `categoryId` is null and
  `isSplit` is true, so the direct-contribution filter already excludes it.
- Sum with `Money` arithmetic from `Money.zero()`. A category with no matching
  rows yields `Money.zero()`.
- `MonthKey.contains(DateTime)` (M1-T02) is the month-membership test — do not
  re-parse dates by hand.

## Tests to write
File: `test/domain/budgeting/category_activity_calculator_test.dart`
Build data with `BudgetFixture` (M2-T01).
- A category with one outflow in the month → activity equals that (negative)
  amount.
- Two transactions in the same month sum; a transaction in a different month is
  excluded.
- A split transaction with two sub-transactions in different categories: each
  category's activity reflects only its own sub-amount.
- A `deleted` transaction and a `deleted` sub-transaction are both excluded.
- A sub-transaction whose **parent** is deleted is excluded.
- An inflow (positive amount) categorised to a category produces positive
  activity.
