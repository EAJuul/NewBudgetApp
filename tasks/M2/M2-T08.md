## Public API to produce
```dart
// added to BudgetService (lib/domain/budgeting/budget_service.dart)
/// Moves [amount] from [fromCategoryId] to [toCategoryId] for [month] by
/// adjusting each category's `assigned` for that month. Total assigned is
/// unchanged, so Ready-to-Assign is unaffected.
Future<void> moveMoney({
  required String fromCategoryId,
  required String toCategoryId,
  required MonthKey month,
  required Money amount,
});
```

## Implementation notes
- For each of the two categories, read the existing `(category, month)`
  assignment via `CategoryBudgetRepository.find`; if none exists, treat the
  current `assigned` as `Money.zero()` and create a new `CategoryBudget` (a
  fresh UUID `id`, via `package:uuid`).
- `from`'s new `assigned = oldFrom - amount`; `to`'s new
  `assigned = oldTo + amount`. Persist both with `CategoryBudgetRepository.save`.
- `assigned` is allowed to go **negative** (`docs/04` invariant #4) — do not
  clamp. Moving more than a category holds simply leaves it negative.
- `amount` is expected to be positive (the direction is the two id arguments);
  a non-positive `amount` is a caller error — assume callers pass a positive
  `Money` (validation belongs in the controller, `docs/09`).
- `moveMoney` writes only — it does not return a `MonthBudget`. Callers re-run
  `computeMonth` (or rely on Drift's reactive streams) to see the result.
- Keep the method on `BudgetService`; do not add a new class.

## Tests to write
File: `test/domain/budgeting/budget_service_move_money_test.dart`
Build a budget with `BudgetFixture`; construct `BudgetService` from its
repositories.
- Move between two categories that both already have an assignment for the
  month: each `assigned` changes by exactly `amount`; `computeMonth`'s
  `readyToAssign` is unchanged.
- Move into a category with **no** prior assignment row for that month: a new
  `CategoryBudget` is created with `assigned == amount`.
- Move more than the source holds: the source's `assigned` goes negative.
- The move affects only the specified `month` — another month's assignments are
  untouched.
