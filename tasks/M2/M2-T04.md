## Public API to produce
```dart
// lib/domain/budgeting/category_available_calculator.dart
/// The `available` balance of a single category for every month in [months].
///
/// [months] must be sorted ascending and contiguous (each the `next()` of the
/// previous). `assignedFor` / `activityFor` supply the per-month assigned
/// amount (from `category_budgets`) and activity (M2-T03).
///
/// Simple rollover: `available(m) = available(m-1) + assigned(m) + activity(m)`,
/// with `available` before the first month treated as zero. All carryover —
/// including a negative balance — rolls forward unchanged. The cash-vs-credit
/// distinction is deferred to M2-T12.
Map<MonthKey, Money> computeCategoryAvailableSeries({
  required List<MonthKey> months,
  required Money Function(MonthKey month) assignedFor,
  required Money Function(MonthKey month) activityFor,
});
```

## Implementation notes
- Iterate `months` in order, carrying a running `Money` balance that starts at
  `Money.zero()`.
- For each month: `running = running + assignedFor(m) + activityFor(m)`; store
  `running` as that month's `available`.
- Return a `Map<MonthKey, Money>` with one entry per month in `months`.
- **Simple mode only.** Do not branch on cash vs credit and do not clamp
  negative carryover to zero — that is M2-T12's job. A negative `available`
  rolls forward verbatim.
- If `months` is empty, return an empty map. Assume `months` is already sorted
  and contiguous — this is internal engine input, not user input, so no
  validation is required (`docs/09` — validate only at boundaries).
- `assigned` may be negative; `activity` is usually negative — both flow through
  the arithmetic unchanged.

## Tests to write
File: `test/domain/budgeting/category_available_calculator_test.dart`
Use plain `assignedFor` / `activityFor` closures (or `BudgetFixture`-derived
data) with hand-computed expected milliunit values.
- One month: `available == assigned + activity`.
- Three contiguous months: month 3's `available` includes months 1–2 carryover.
- A month with `assigned` only (no activity), and a month with activity only.
- Negative carryover: a month that overspends leaves a negative `available`
  that rolls into the next month (simple-mode behaviour).
- A negative `assigned` (money pulled back to RTA) reduces `available`.
- Empty `months` → empty map.
