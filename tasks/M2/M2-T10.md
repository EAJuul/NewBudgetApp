## Public API to produce
```dart
// lib/domain/budgeting/credit_card_payment_category.dart
/// Builds the payment [Category] for the credit-card [account].
///
/// The category is linked back to the account via `linkedAccountId` and lives
/// in the `creditCardPayments` system group identified by [paymentGroupId].
/// [categoryId] is a caller-generated UUID. Throws [ArgumentError] when
/// [account] is not a credit-card account.
Category creditCardPaymentCategory({
  required Account account,
  required String paymentGroupId,
  required String categoryId,
  required int sortOrder,
});
```

## Implementation notes
- Guard first: if `!account.type.isCreditCard`, throw an `ArgumentError` — only
  `AccountType.creditCard` / `lineOfCredit` accounts get a payment category.
  (`AccountType.isCreditCard` is true only for `creditCard`; treat
  `lineOfCredit` per `account.type.isCreditCard` — if it must also qualify,
  that is a domain decision; **NEEDS REVIEW:** confirm whether `lineOfCredit`
  gets a payment category. Until confirmed, accept exactly what
  `AccountType.isCreditCard` reports.)
- Build and return:
  `Category(id: categoryId, groupId: paymentGroupId, name: account.name,
  hidden: false, note: null, sortOrder: sortOrder, linkedAccountId: account.id)`.
- The resulting `Category.isCreditCardPayment` getter (M1-T15) is therefore
  `true`, because `linkedAccountId` is non-null.
- The function is pure — it builds an entity, it does not persist anything. The
  caller (M3) saves it through `CategoryRepository` inside the account-create
  transaction.
- This task does **not** create the `creditCardPayments` system group; that
  group is seeded by onboarding (M10). The caller passes its id.

## Tests to write
File: `test/domain/budgeting/credit_card_payment_category_test.dart`
- For an `AccountType.creditCard` account, the returned `Category` has
  `linkedAccountId == account.id`, `name == account.name`,
  `groupId == paymentGroupId`, and `isCreditCardPayment == true`.
- The returned category uses the supplied `categoryId` and `sortOrder`.
- For an `AccountType.checking` account, the function throws `ArgumentError`.
- For an `AccountType.asset` account, the function throws `ArgumentError`.
