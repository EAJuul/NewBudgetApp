## Public API to produce
```dart
// lib/domain/budgeting/budget_service.dart
class BudgetService {
  BudgetService({
    required AccountRepository accountRepository,
    required CategoryRepository categoryRepository,
    required CategoryBudgetRepository categoryBudgetRepository,
    required TransactionRepository transactionRepository,
  });

  /// The fully computed [MonthBudget] for [month] in [budgetId].
  Future<MonthBudget> computeMonth(String budgetId, MonthKey month);
}
```

## Implementation notes
`computeMonth` steps:
1. **Load accounts** — `await accountRepository.watchAll(budgetId).first`.
2. **Load categories** — `groups = await categoryRepository.watchAllGroups(
   budgetId).first`; for each group
   `await categoryRepository.watchCategoriesInGroup(group.id).first`; flatten to
   one `List<Category>`.
3. **Load transactions** — `await transactionRepository.allForBudget(budgetId)`.
   For every transaction with `isSplit == true`, also fetch
   `transactionRepository.subTransactionsOf(id)` and collect a combined
   `List<SubTransaction>`.
4. **Load assignments** — per category,
   `await categoryBudgetRepository.watchForCategory(category.id).first`.
5. **Month range** — the series starts at the earliest `MonthKey` present in any
   assignment or any transaction date (`MonthKey.fromDate`), and runs through
   `month`. Build the contiguous ascending `List<MonthKey>` with
   `MonthKey.next()`. If there is no data at all, the range is just `[month]`.
6. **Per category** — compute `activity` for each month with
   `computeCategoryActivity` (M2-T03); take `assigned` from the assignments;
   feed both as closures to `computeCategoryAvailableSeries` (M2-T04); read the
   line for `month` → a `CategoryBudgetLine` of (assigned, activity, available).
7. **Ready to Assign** — `computeReadyToAssign` (M2-T05) over all accounts, all
   transactions, all assignments.
8. Return `MonthBudget(month: month, readyToAssign: rta, lines: lines)`.
- Use the **simple** rollover (M2-T04) — the cash/credit rule is wired in later
  by M2-T12; do not branch on account type here.
- The service consumes repository **interfaces** only — it never imports a
  `*_impl.dart`, a DAO, or `package:drift`.
- `computeMonth` is a pure read: it performs no writes.

## Tests to write
File: `test/domain/budgeting/budget_service_test.dart`
Build a known budget with `BudgetFixture` (M2-T01); construct `BudgetService`
from the fixture's repositories.
- An empty budget → `MonthBudget` with `readyToAssign == zero` and no lines (or
  zero-valued lines if categories exist).
- One account + one funded category + one outflow: assert the category line's
  `assigned`, `activity`, `available`, and the month's `readyToAssign` against
  hand-computed milliunits.
- Rollover: assign in month 1, view month 2 → month 2's `available` includes the
  month-1 carryover.
- A split transaction contributes to the right category lines.
- `computeMonth` for a month with no data still returns a valid `MonthBudget`.
