## Public API to produce
```dart
// lib/domain/budgeting/account_balance_calculator.dart
@freezed
class AccountBalances with _$AccountBalances {
  const factory AccountBalances({
    required Money working,
    required Money cleared,
    required Money uncleared,
  }) = _AccountBalances;
}

/// Balances for [accountId], computed from [transactions] (any mix of accounts
/// may be passed — the function filters by `accountId`).
AccountBalances computeAccountBalances({
  required String accountId,
  required Iterable<Transaction> transactions,
});
```

## Implementation notes
- Consider only transactions whose `accountId` matches and whose `deleted` is
  `false`.
- **Working balance** = the sum of those transactions' `amount`s.
- **Cleared balance** = the same sum restricted to
  `cleared == ClearedStatus.cleared || cleared == ClearedStatus.reconciled`.
- **Uncleared balance** = `working - cleared`.
- Sum with `Money` arithmetic starting from `Money.zero()` — never `double`,
  never `int` outside `Money` (`docs/09`).
- An account with no transactions yields all-`zero` balances.
- Split parents: a split parent's `amount` equals the sum of its children
  (`docs/04` invariant #2), so summing parent `amount`s is correct — do **not**
  also add `sub_transactions`, that would double-count.
- `AccountBalances` is `freezed`; run `build_runner` for the generated part.

## Tests to write
File: `test/domain/budgeting/account_balance_calculator_test.dart`
Build data with `BudgetFixture` (M2-T01).
- An account with no transactions → working / cleared / uncleared all zero.
- Mixed `uncleared` and `cleared` transactions: assert each balance against a
  hand-computed milliunit total.
- A `reconciled` transaction counts toward the cleared balance.
- A `deleted` transaction is excluded from all three balances.
- Transactions for a different account are excluded.
- A negative working balance (outflows exceed inflows) is reported correctly.
