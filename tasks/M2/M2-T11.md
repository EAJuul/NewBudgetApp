## Public API to produce
```dart
// lib/domain/budgeting/transfer.dart
/// The net milliunit amount transfers moved into [accountId]: the sum of the
/// signed `amount`s of that account's non-deleted transfer rows. Positive means
/// transfers brought money in; negative means they took money out.
Money netTransferAmount({
  required String accountId,
  required Iterable<Transaction> transactions,
});
```

## Implementation notes
- A transfer is two transaction rows, one per account, holding equal-and-opposite
  `amount`s (`docs/04` invariant #3). Each row is a normal transaction on its
  account, so `computeAccountBalances` (M2-T02) **already** includes transfers
  in the working/cleared balances — this task does not change that calculator.
- `netTransferAmount` sums `amount` over `transactions` where
  `accountId` matches, `deleted == false`, and `isTransfer == true` (the
  `Transaction.isTransfer` getter, M1-T19). Use `Money` arithmetic from
  `Money.zero()`.
- The helper exists so callers that need transfers separated from spending
  (net-worth, M3-T12; reconciliation, M10) do not re-derive the filter.
- A transfer between two on-budget accounts nets to zero across the pair but is
  non-zero for each individual account — that is correct and expected.

## Tests to write
File: `test/domain/budgeting/transfer_test.dart`
Build data with `BudgetFixture` (M2-T01); create transfer pairs by giving two
transactions matching `transferTransactionId` / `transferAccountId` and
equal-and-opposite amounts.
- `netTransferAmount` for an account with one inbound transfer equals that
  positive amount; with one outbound transfer, the negative amount.
- Inbound and outbound transfers on the same account sum correctly.
- A `deleted` transfer row is excluded.
- A non-transfer transaction does not affect `netTransferAmount`.
- `computeAccountBalances` (M2-T02): an account receiving a transfer has its
  working balance increased by the transfer amount; the sending account is
  decreased by the same amount.
- A transfer marked `cleared` contributes to the cleared balance; an `uncleared`
  one does not.
