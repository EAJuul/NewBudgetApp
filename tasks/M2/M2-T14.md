## Public API to produce
```dart
// lib/domain/budgeting/schedule.dart
/// The next occurrence strictly after [date] for a schedule of [frequency].
/// [date] is treated as a calendar date (the time component is ignored;
/// the result has a zero time component).
DateTime advance(DateTime date, ScheduleFrequency frequency);
```

## Implementation notes
- Normalise to a date-only `DateTime(year, month, day)` on entry and return.
- Frequency rules:
  - `daily` → `+1 day`
  - `weekly` → `+7 days`
  - `everyOtherWeek` → `+14 days`
  - `every4Weeks` → `+28 days`
  - `monthly` → `+1 month`
  - `everyOtherMonth` → `+2 months`
  - `every3Months` → `+3 months`
  - `every6Months` → `+6 months`
  - `yearly` → `+12 months`
  - `twiceAMonth` → if `day < 15`, the 15th of the same month; otherwise the
    1st of the next month. **NEEDS REVIEW:** confirm the intended semi-monthly
    anchor days (15th / 1st assumed here) before M4-T20.
- **Month arithmetic must clamp the day.** Adding months to Jan 31 must yield
  the last valid day of the target month (Feb 28, or Feb 29 in a leap year),
  not overflow into the following month. Compute the target `(year, month)`,
  then clamp the day to that month's length. Do not rely on
  `DateTime(y, m + n, day)` overflow behaviour — it rolls over instead of
  clamping.
- Day-based frequencies (`daily`..`every4Weeks`) need no clamping —
  `DateTime.add(Duration(days: n))` is correct, including across DST since the
  input is date-only.
- The function is total: every `ScheduleFrequency` member is handled (a
  `switch` with no `default`, so a future enum member is a compile error).

## Tests to write
File: `test/domain/budgeting/schedule_test.dart`
Exhaustive case table (`docs/07` — calendar edge cases mandatory).
- Each day-based frequency advances by the right number of days, including
  across a month boundary and a year boundary.
- `monthly` from Jan 15 → Feb 15; from Dec 15 → Jan 15 next year.
- `monthly` from **Jan 31** → **Feb 28** in a common year and **Feb 29** in a
  leap year (e.g. 2028).
- `monthly` from Mar 31 → Apr 30 (clamp to a 30-day month).
- `everyOtherMonth`, `every3Months`, `every6Months`, `yearly` cross year
  boundaries and clamp correctly (e.g. `yearly` from Feb 29 2028 → Feb 28 2029).
- `twiceAMonth` from the 5th → the 15th; from the 20th → the 1st of next month.
- The returned `DateTime` always has a zero time component.
