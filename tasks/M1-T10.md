## Public API to produce
```dart
// lib/data/database/tables/scheduled_transactions_table.dart
@DataClassName('ScheduledTransactionRow')
class ScheduledTransactions extends Table {
  TextColumn get id => text()();
  TextColumn get accountId => text()();
  IntColumn get amount => integer()();                       // signed milliunits
  TextColumn get payeeId => text().nullable()();
  TextColumn get categoryId => text().nullable()();
  TextColumn get memo => text().nullable()();
  TextColumn get frequency => textEnum<ScheduleFrequency>()();
  TextColumn get nextDate => text()();                       // 'YYYY-MM-DD'
  TextColumn get createdAt => text()();
  TextColumn get updatedAt => text()();

  @override
  Set<Column> get primaryKey => {id};
}
```

## Implementation notes
- **`@DataClassName('ScheduledTransactionRow')` is required** so the generated
  row class does not collide with the domain entity `ScheduledTransaction`
  (M1-T24). Every table in this schema uses the `<Entity>Row` convention.
- `frequency` is persisted by `.name` via `textEnum<ScheduleFrequency>()`. Do
  not reorder or rename the enum members without a schema migration.
- `nextDate` is the next occurrence still to materialize, ISO `YYYY-MM-DD` text.
- `amount` is signed milliunits, like `transactions.amount`.
- MVP scheduled transactions are single-category — there is no split support on
  this table (`docs/03`).
- `accountId`, `payeeId`, `categoryId` are logical FKs to tables owned by other
  cards. Leave them as plain `text()` / `text().nullable()` columns — M1-T11
  adds the FK constraints centrally. **NEEDS REVIEW:** M1-T11.
- Do **not** add this table to `AppDatabase.tables` — that is M1-T11.

## Tests to write

Test cases:
- Insert a `ScheduledTransactionsCompanion` with
  `frequency: ScheduleFrequency.monthly`; select it back and verify `frequency`
  and `nextDate`.
- Insert a row for `ScheduleFrequency.everyOtherWeek` and one for
  `ScheduleFrequency.yearly`; verify `frequency` round-trips via `textEnum` for
  each.
- Insert a row with `payeeId` and `categoryId` both null; read it back and
  assert both are null.
- Insert a row with a negative `amount`; verify it round-trips.
