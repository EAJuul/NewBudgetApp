## Public API to produce
```dart
// lib/data/database/tables/payees_table.dart
class Payees extends Table {
  TextColumn get id => text()();
  TextColumn get budgetId => text()();
  TextColumn get name => text()();
  // FK→categories.id; null when no default category is set
  TextColumn get defaultCategoryId => text().nullable()();
  // FK→accounts.id; set when this payee represents "Transfer: <account>"
  TextColumn get transferAccountId => text().nullable()();

  @override
  Set<Column> get primaryKey => {id};
}

// lib/data/database/tables/settings_table.dart
class Settings extends Table {
  // The setting key IS the primary key — no separate UUID `id` column.
  TextColumn get key => text()();
  TextColumn get value => text()();

  @override
  Set<Column> get primaryKey => {key};
}
```

## Implementation notes
- `defaultCategoryId` (FK→`categories.id`) and `transferAccountId` (FK→`accounts.id`)
  are both nullable text columns. Define them without `.references()` — the
  referenced table classes (M1-T05, M1-T06) may not be complete when M1-T08 is
  worked. **NEEDS REVIEW:** add FK constraints during M1-T11.
- `Settings` uses a `TEXT` primary key (`key`) — no UUID, no separate `id`
  column. This is the only table in the schema that deviates from the UUID-PK
  convention; it is intentional (key-value store).
- `Payees` has no `created_at` / `updated_at` columns per the data model.
- Do **not** add these tables to `AppDatabase.tables` — that is M1-T11.

## Tests to write
File: `test/data/database/tables/payees_settings_test.dart`

```dart
@DriftDatabase(tables: [Payees, Settings])
class _TestDb extends _$_TestDb {

```

Test cases:
- Insert a `PayeesCompanion` with both nullable columns null; read it back and
  assert both are null.
- Insert a payee with a non-null `transferAccountId`; verify the text value
  round-trips correctly.
- Insert two `SettingsCompanion` rows with distinct keys; verify both are
  selectable by key.
- Insert a settings row, then insert again with the same key (or use an upsert);
  verify the value is updated and only one row with that key exists.
