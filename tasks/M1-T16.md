## Public API to produce
```dart
// lib/data/daos/categories_dao.dart
import 'package:budget_app/data/database/app_database.dart';
import 'package:budget_app/data/database/tables/categories_table.dart';
import 'package:budget_app/data/database/tables/category_groups_table.dart';
import 'package:drift/drift.dart';

part 'categories_dao.g.dart';

@DriftAccessor(tables: [CategoryGroups, Categories])
class CategoriesDao extends DatabaseAccessor<AppDatabase>
    with _$CategoriesDaoMixin {
  CategoriesDao(super.db);

  /// Groups in [budgetId], ordered by `sortOrder`, re-emitted on change.
  Stream<List<CategoryGroupRow>> watchGroupsByBudget(String budgetId);

  /// Categories in [groupId], ordered by `sortOrder`, re-emitted on change.
  Stream<List<CategoryRow>> watchCategoriesInGroup(String groupId);

  Future<CategoryRow?> findCategoryById(String id);

  Future<void> upsertGroup(CategoryGroupsCompanion group);
  Future<void> upsertCategory(CategoriesCompanion category);

  Future<void> deleteGroup(String id);
  Future<void> deleteCategory(String id);
}
```

## Implementation notes
- One DAO covers both tables because they are a single aggregate (a group owns
  its categories). It is a standalone `@DriftAccessor` constructed as
  `CategoriesDao(db)`. Run `build_runner` for `_$CategoriesDaoMixin`.
- Generated types: rows `CategoryGroupRow` / `CategoryRow`, companions
  `CategoryGroupsCompanion` / `CategoriesCompanion` (see M1-T06).
- Both `watch*` queries `..orderBy([(t) => OrderingTerm(expression: t.sortOrder)])`
  and end in `.watch()`.
- `upsert*` use `insertOnConflictUpdate`.
- The DAO returns Drift row types only; mapping to domain `CategoryGroup` /
  `Category` is the repository's job (M1-T17).

## Tests to write
File: `test/data/daos/categories_dao_test.dart`

Real in-memory database, closed in `tearDown`. `PRAGMA foreign_keys = ON` is
active (M1-T11): a `categories` row needs an existing `category_groups` parent,
which needs a `budgets` parent — seed `budgets` then `category_groups` first.

Test cases:
- `upsertGroup` a group, then `watchGroupsByBudget` emits it; a group from a
  different budget is excluded.
- `upsertCategory` two categories in one group; `watchCategoriesInGroup` emits
  them ordered by `sortOrder`.
- `upsertCategory` again with an existing `id` and a changed `name`;
  `findCategoryById` shows the update and no duplicate row appears.
- `deleteCategory` removes a category; `findCategoryById` returns `null`.
- `deleteGroup` removes a group.
