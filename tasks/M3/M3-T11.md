## Goal
A widget that presents accounts grouped into three sections — Budget,
Tracking, and Closed — the way a YNAB-style sidebar does. Intended for the
navigation drawer / wide-screen side panel.

/// Accounts grouped into Budget / Tracking / Closed sections, each section a
/// labelled header followed by its account rows. Empty sections are omitted.
class AccountGroups extends ConsumerWidget {
  const AccountGroups({this.onAccountTap, super.key});

  /// Invoked with the tapped account's id (e.g. for navigation).
  final void Function(String accountId)? onAccountTap;

  Widget build(BuildContext context, WidgetRef ref);

## Implementation notes
- Watch `accountListControllerProvider` and switch on the `AsyncValue` with
  `.when` (loading / error / data) — same pattern as M3-T02.
- Partition the `data` list into three groups:
  - **Budget** — `!closed && account.type.isOnBudget`.
  - **Tracking** — `!closed && !account.type.isOnBudget`.
  - **Closed** — `closed` (regardless of type).
- Render each non-empty group as a section header (`Text` with a heading
  style) followed by one `AccountListItem` per account, passing
  `onTap: () => onAccountTap?.call(account.id)`.
- Omit a section header entirely when its group is empty.
- Do **not** compute balances or section totals here — each `AccountListItem`
  shows its own balance; a budget-wide total is the separate net-worth widget
  (M3-T12).
- Use a scrollable column (`ListView` / `SingleChildScrollView`) so long
  account lists do not overflow.
