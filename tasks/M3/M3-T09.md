## Public API to produce
```dart
// lib/features/accounts/presentation/account_detail_controller.dart
part 'account_detail_controller.freezed.dart';
part 'account_detail_controller.g.dart';

/// An account plus its derived balances — the detail screen's view model.
@freezed
class AccountDetail with _$AccountDetail {
  const factory AccountDetail({
    required Account account,
    required AccountBalances balances,
  }) = _AccountDetail;
}

/// Streams the [AccountDetail] for [accountId]; balances recompute whenever
/// the account's transactions change.
@riverpod
class AccountDetailController extends _$AccountDetailController {
  @override
  Stream<AccountDetail> build(String accountId);
}
```

## Implementation notes
- `build` is an `async*` generator:
  1. `final account = await ref.watch(accountRepositoryProvider).findById(accountId);`
  2. if `account == null`, `throw StateError('Account not found: $accountId')`
     — the screen renders the resulting `AsyncError`.
  3. `yield*` `transactionRepository.watchByAccount(accountId)` mapped to
     `AccountDetail(account: account, balances: computeAccountBalances(
     accountId: accountId, transactions: txns))`.
- The account entity is fetched once; balances are the reactive part. An edit
  to the account's name (rare, and it routes through its own screen) is not
  expected to live-update this screen — that is acceptable for the MVP.
- `AccountDetail` is `freezed`; run `build_runner` for both the `.freezed.dart`
  and the `.g.dart` parts.
- `build` takes `accountId` as a parameter — Riverpod generates a family
  provider `accountDetailControllerProvider(accountId)`.

## Tests to write
File: `test/features/accounts/presentation/account_detail_controller_test.dart`

`ProviderContainer` with `appDatabaseProvider` overridden to an in-memory
database; seed a `Budget` and accounts/transactions through the repository
providers.
- An account with no transactions → `balances` all zero, `account` matches.
- After seeding cleared and uncleared transactions, `balances.working`,
  `.cleared`, and `.uncleared` equal hand-computed milliunit totals.
- Inserting another transaction makes the controller emit updated balances.
- An unknown `accountId` surfaces an `AsyncError`.
