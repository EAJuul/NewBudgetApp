## Goal
The reusable list-row widget for one account: its name and its working
balance. The widget fetches its own balance, so any list (M3-T02 flat list,
M3-T11 grouped sidebar) can drop it in.

## Public API to produce
```dart
// lib/features/accounts/presentation/widgets/account_list_item.dart
part 'account_list_item.g.dart';

/// The working balance of [accountId] — recomputes when its transactions
/// change. A family provider so each row owns its own balance subscription.
@riverpod
Stream<Money> accountWorkingBalance(Ref ref, String accountId);

/// A single account row: name on the left, working balance on the right.
class AccountListItem extends ConsumerWidget {
  const AccountListItem({required this.account, this.onTap, super.key});

  final Account account;
  final VoidCallback? onTap;

  @override
  Widget build(BuildContext context, WidgetRef ref);
}
```

## Implementation notes
- `accountWorkingBalance` returns
  `transactionRepository.watchByAccount(accountId)` mapped through
  `computeAccountBalances(accountId: accountId, transactions: txns).working`.
  This is the one place balances are computed for the list — keep the screen
  (M3-T02) and the grouped widget (M3-T11) free of balance logic.
- `AccountListItem` builds a `ListTile`:
  - `title` → `account.name`.
  - `trailing` → the balance. Watch
    `accountWorkingBalanceProvider(account.id)`; on `data` show
    `CurrencyFormatter.format(balance)`, on `loading` show a small neutral
    placeholder (e.g. an em dash), on `error` show a short error glyph/text.
  - When `account.closed`, show a subtle "Closed" marker (e.g. the `subtitle`
    or a muted text style) — do not hide closed accounts.
  - `onTap` → forwarded straight to `ListTile.onTap`.
- Riverpod generates `account_list_item.g.dart` — run `build_runner`.
- No business logic beyond reading the provider; the widget stays dumb.
