## Public API to produce
```dart
// added to class AccountListController in account_list_controller.dart
/// Sets the account's `closed` flag. The watched list updates automatically.
Future<void> setClosed(String accountId, {required bool closed});
```

## Implementation notes
- `setClosed` reads `accountRepositoryProvider`, `findById(accountId)`; if the
  account is missing, throw a `StateError`. Otherwise
  `accountRepository.save(account.copyWith(closed: closed))`.
- No manual `state` update is needed — `build` already streams `watchAll`, so
  the closed-flag change propagates to every listener automatically.
- Closing an account does **not** delete it, change its transactions, or touch
  any credit-card category — keep the method to the single flag flip.
- **UI control** in `edit_account_screen.dart`: add one button below the
  existing form — labelled "Close account" when the account is open and
  "Reopen account" when it is closed. On press, call
  `ref.read(accountListControllerProvider.notifier)
  .setClosed(accountId, closed: !account.closed)`, then show a confirmation
  `SnackBar`. Closing should prompt for confirmation first (an
  `AlertDialog`); reopening may act directly.
- Do not add a delete control — that is M3-T08.

## Tests to write
File: `test/features/accounts/presentation/account_close_reopen_test.dart`

`ProviderContainer` with `appDatabaseProvider` overridden to an in-memory
database; seed a `Budget` and an account.
- `setClosed(id, closed: true)` flips the account's `closed` flag to `true`;
  re-reading it through the repository confirms the change.
- `setClosed(id, closed: false)` on a closed account reopens it.
- The account stays present in `AccountListController`'s list after closing
  (it is hidden by grouping, not removed).
- `setClosed` on an unknown id throws a `StateError`.
