## Goal
The Riverpod controller behind the account list screen: it resolves the
primary budget and streams that budget's accounts, sorted for display. This is
the first presentation-layer file in the project.

## Public API to produce
```dart
// lib/features/accounts/presentation/account_list_controller.dart
part 'account_list_controller.g.dart';

/// Streams every account of the primary budget, sorted by `sortOrder`
/// then `name`. Closed accounts are included — grouping is the caller's job.
@riverpod
class AccountListController extends _$AccountListController {
  @override
  Stream<List<Account>> build();
}
```

## Implementation notes
- `build` is an `async*` generator:
  1. `final budget = await ref.watch(budgetRepositoryProvider).getPrimary();`
  2. if `budget == null`, `yield const <Account>[];` and `return` — onboarding
     has not created a budget yet.
  3. otherwise `yield*` `accountRepository.watchAll(budget.id)` mapped to a
     **sorted copy**: `[...accounts]..sort(_bySortOrderThenName)`.
- Sort comparator: compare `sortOrder` ascending; break ties on `name`
  (case-insensitive). Keep it a private top-level function.
- Do not compute balances here — that is the list item widget's job (M3-T03).
  This controller deals only in `Account` entities.
- Do not add `close` / `reopen` / `delete` methods — those are added later by
  M3-T07 and M3-T08. Deliver only `build` in this task.
- Riverpod generates `account_list_controller.g.dart` — run `build_runner`.
- Consumers read `AsyncValue<List<Account>>` via
  `ref.watch(accountListControllerProvider)`.

## Tests to write
File: `test/features/accounts/presentation/account_list_controller_test.dart`

Build a `ProviderContainer` overriding `appDatabaseProvider` with an in-memory
database (the M1-T26 test pattern); seed data through the repository providers
read from that same container.
- With no `Budget` seeded, the controller's first value is an empty list.
- With a `Budget` but no accounts, the value is an empty list.
- With three accounts seeded out of order, the emitted list is sorted by
  `sortOrder` then `name`.
- Closed accounts appear in the list (they are not filtered out).
- Inserting another account through `accountRepositoryProvider` makes the
  controller emit an updated list (reactive stream).
