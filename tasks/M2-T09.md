## Public API to produce
```dart
// added to BudgetService (lib/domain/budgeting/budget_service.dart)
/// Sets the `assigned` amount of [categoryId] for [month] to the absolute
/// value [amount] (not a delta). Creates the assignment row if none exists.
Future<void> setAssigned({
  required String categoryId,
  required MonthKey month,
  required Money amount,
});
```

## Implementation notes
- Read the existing `(category, month)` assignment via
  `CategoryBudgetRepository.find`.
  - If it exists, save a copy with `assigned` replaced by `amount`
    (`copyWith(assigned: amount)`).
  - If not, create a new `CategoryBudget` with a fresh UUID `id` (via
    `package:uuid`), the given `categoryId` / `month`, and `assigned == amount`.
- This is an **absolute set**, not an increment — contrast with M2-T08's
  `moveMoney`. Setting `amount` smaller than before reduces total assigned and
  raises Ready-to-Assign; setting it larger lowers RTA.
- `amount` may be negative — assigning a negative value pulls money back to RTA
  (`docs/04` invariant #4). Do not clamp or reject it.
- `setAssigned` writes only; it returns nothing. Callers re-run `computeMonth`
  (or rely on reactive streams) to observe the change.
- Keep the method on `BudgetService`; add no new class.

## Tests to write
File: `test/domain/budgeting/budget_service_set_assigned_test.dart`
Build a budget with `BudgetFixture`; construct `BudgetService` from its
repositories.
- `setAssigned` on a category with no prior row creates one; `computeMonth`
  shows that `assigned` and a correspondingly reduced `readyToAssign`.
- `setAssigned` over an existing assignment replaces (does not add to) the
  value.
- `setAssigned` to `Money.zero()` zeroes the category and returns that money to
  RTA.
- `setAssigned` to a negative amount is stored verbatim and raises RTA.
- Setting one month does not change another month's assignment.
