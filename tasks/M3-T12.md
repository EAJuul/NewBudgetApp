## Implementation notes
- Watch `accountListControllerProvider` for the account list. While it is
  loading or errored, render the matching state (a small placeholder /
  message).
- For the `data` list, take only **non-closed** accounts. For each, watch
  `accountWorkingBalanceProvider(account.id)` — calling `ref.watch` inside a
  loop over the accounts is fine in a `ConsumerWidget` build.
- Reuse the M3-T03 balance provider rather than recomputing balances — this
  keeps the total reactive: any transaction change in any account re-runs that
  account's balance provider and rebuilds this widget.
- Sum the resolved balances with `Money` arithmetic starting from
  `Money.zero()`. If any account's balance provider is still loading, show a
  loading placeholder; if any errored, show an error placeholder; only show
  the total once every balance has resolved.
- Display: a label ("Net Worth") and the total formatted via
  `CurrencyFormatter.format`.
- No `double`, no `int` outside `Money`.
