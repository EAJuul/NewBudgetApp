## Public API to produce
```dart
// lib/data/database/tables/category_groups_table.dart
class CategoryGroups extends Table {
  TextColumn get id => text()();
  TextColumn get budgetId => text()();
  TextColumn get name => text()();
  BoolColumn get hidden => boolean()();
  IntColumn get sortOrder => integer()();
  // null for normal groups; non-null for system groups
  TextColumn get systemType => textEnum<SystemGroupType>().nullable()();

  @override
  Set<Column> get primaryKey => {id};
}

// lib/data/database/tables/categories_table.dart
class Categories extends Table {
  TextColumn get id => text()();
  TextColumn get groupId => text()();
  TextColumn get name => text()();
  BoolColumn get hidden => boolean()();
  TextColumn get note => text().nullable()();
  IntColumn get sortOrder => integer()();
  // set only for a credit-card payment category; FK to accounts.id
  TextColumn get linkedAccountId => text().nullable()();
  TextColumn get createdAt => text()();
  TextColumn get updatedAt => text()();

  @override
  Set<Column> get primaryKey => {id};
}
```

## Implementation notes
- `systemType` is nullable: regular groups leave it null. Use
  `textEnum<SystemGroupType>().nullable()()`.
- `linkedAccountId` in `Categories` is a logical FK to `accounts.id` (only set
  for credit-card payment categories). Define it as plain `text().nullable()()`
  without `.references()` — the `Accounts` table class (M1-T05) may not be
  complete when M1-T06 is worked. **NEEDS REVIEW:** add the FK reference
  constraint during M1-T11 when all tables are wired together.
- `groupId` in `Categories` references `CategoryGroups.id`. Use
  `.references(CategoryGroups, #id)` since both tables are in this task.
- `category_groups` has no `created_at` / `updated_at` columns per the data
  model — do not add them.
- Do **not** add these tables to `AppDatabase.tables` — that is M1-T11.

## Tests to write
File: `test/data/database/tables/category_groups_categories_test.dart`

Define a test-only database in the test file:
```dart
@DriftDatabase(tables: [CategoryGroups, Categories])
class _TestDb extends _$_TestDb {
  _TestDb() : super(NativeDatabase.memory());
  @override
  int get schemaVersion => 1;
}
```

Test cases:
- Insert a `CategoryGroupsCompanion` with `systemType: Value(null)` and read it
  back; assert `systemType` is null.
- Insert a group with `systemType: Value(SystemGroupType.creditCardPayments)`;
  assert it round-trips via `textEnum`.
- Insert a `CategoriesCompanion` with `linkedAccountId: Value(null)`; read it
  back and confirm null.
- Insert a category with a valid `groupId` and `hidden: Value(false)`; verify
  both values are stored correctly.
