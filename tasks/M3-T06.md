## Implementation notes
- On init, load the account once: `accountRepositoryProvider`'s `findById`
  in a `FutureProvider`/`FutureBuilder`, or `await` it in `initState` and hold
  it in state. Show a progress indicator until it resolves; show an error
  state if the account is not found.
- Pre-fill a `Form` with the loaded account: name `TextFormField`, type
  `DropdownButtonFormField<AccountType>`, and an optional note
  `TextFormField` (may be empty → `null`). Reuse the `_label(AccountType)`
  display mapping style from M3-T05 (a private function in this file — do not
  share code across screen files for the MVP).
- Save:
  1. validate the `Form`; abort if invalid.
  2. `await ref.read(accountFormControllerProvider.notifier).update(
     accountId: widget.accountId, name: …, type: …, note: …)`.
  3. on success navigate back — `context.go('/accounts/${widget.accountId}')`.
  4. on failure show a `SnackBar`.
- Watch `accountFormControllerProvider` to disable Save and show progress
  while loading.
- The starting balance is **not** editable here — it is a transaction, not an
  account field.
- Dispose every `TextEditingController`.
