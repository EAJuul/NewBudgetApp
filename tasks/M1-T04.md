# M1-T04 — `AppDatabase` skeleton

- **Milestone:** M1
- **Dependencies:** M1-T03

## Goal
Stand up the Drift database class with a connection, a schema version, a
testing constructor, and a Riverpod provider. Table definitions are added later
and wired in by M1-T11.

## Public API to produce
```dart
// app_database.dart
@DriftDatabase(tables: [])           // the table list is populated in M1-T11
class AppDatabase extends _$AppDatabase {
  AppDatabase() : super(openConnection());
  AppDatabase.forTesting(QueryExecutor executor) : super(executor);

  @override
  int get schemaVersion => 1;
}

@riverpod
AppDatabase appDatabase(Ref ref);    // use the ref type your riverpod_generator
                                     // version expects; closes the db onDispose

// connection.dart
QueryExecutor openConnection();      // a file-backed db in the app docs dir
```

## Implementation notes
- `openConnection()` resolves the application-documents directory with
  `path_provider` and opens a `NativeDatabase` for the file `budget_app.sqlite`.
- `AppDatabase.forTesting` lets tests inject `NativeDatabase.memory()`.
- The `appDatabase` provider must call `ref.onDispose(db.close)`.
- Run `dart run build_runner build --delete-conflicting-outputs` to generate
  `app_database.g.dart`.

## Tests to write
File: `test/data/database/app_database_test.dart`
- An `AppDatabase.forTesting(NativeDatabase.memory())` instance opens and
  reports `schemaVersion == 1`.
- The database closes without error.
