## Goal

starting balance, submit. It delegates persistence to the account form

## Implementation notes
- `ConsumerStatefulWidget` because the form holds ephemeral UI state —
  `TextEditingController`s for name and balance, and the selected
  `AccountType`. Ephemeral form state stays local; only persistence goes
  through the controller.
- `Scaffold` + `AppBar` titled "New Account". Body is a `Form` with:
  - a name `TextFormField` — validator: non-empty after trim.
  - an account-type selector — `DropdownButtonFormField<AccountType>` over
    `AccountType.values`. Map each enum to a human label with a small private
    `_label(AccountType)` function (display-only mapping; lives in this file).
  - a starting-balance `TextFormField` — numeric keyboard; validator: empty is
    allowed (treated as zero) and otherwise must parse via
    `CurrencyFormatter.tryParse`. Default an empty field to `Money.zero()`.
- Submit:
  1. validate the `Form`; abort if invalid.
  2. `await ref.read(accountFormControllerProvider.notifier).create(
     name: …, type: …, startingBalance: …)`.
  3. on success navigate back to the list — `context.go('/accounts')`.
  4. on failure show a `SnackBar` with the error.
- Watch `accountFormControllerProvider` to disable the submit button and show
  a progress indicator while `state.isLoading`.
- Dispose every `TextEditingController` in `dispose()`.
