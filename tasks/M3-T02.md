## Implementation notes
- `Scaffold` with an `AppBar` titled "Accounts".
- Watch `accountListControllerProvider` and switch on the `AsyncValue` with
  `.when`:
  - `loading` → a centered `CircularProgressIndicator`.
  - `error` → a centered error message (plain `Text`; no stack trace).
  - `data` with an empty list → a centered empty state: a short message such
    as "No accounts yet" prompting the user to add one.
  - `data` with accounts → a `ListView` of `AccountListItem`, one per account,
    each with `onTap: () => context.go('/accounts/${account.id}')`.
- A `FloatingActionButton` (or an `AppBar` `+` action) navigates to
  `context.go('/accounts/new')`.
- Use `context.go` from `package:go_router`. The `/accounts/...` routes are
  registered in M3-T13; referencing the path strings here is fine — go_router
  resolves them at runtime.
- The widget is dumb: no business logic, no balance computation. `AccountListItem`
  (M3-T03) fetches each row's balance itself.
- This screen renders the flat list. The grouped (Budget / Tracking / Closed)
  presentation is a separate widget delivered by M3-T11.
