## Implementation notes
- The entity stores `month` as a `MonthKey` and `assigned` as `Money`. The
  mapper converts: row `month` text ↔ `MonthKey.parse` / `.toIso()`; row
  `assigned` int ↔ `Money(...)` / `.milliunits`.
- `assigned` may be **negative** (`docs/04` invariant #4) — no validation.
- `save` preserves `createdAt` across updates, same pattern as M1-T14: look up
  the existing row by `(categoryId, month)`, reuse its `createdAt`.
- The `UNIQUE(category_id, month)` constraint (M1-T07) plus
  `insertOnConflictUpdate` give clean upsert-by-(category, month) semantics even
  though the primary key is `id`. The DAO `upsert` relies on that.
- There is **no delete** — an assignment row, once created, persists; the user
  zeroes it by saving `assigned = Money.zero()` (`docs/03`).
- `CategoryBudgetRepositoryImpl(AppDatabase db)` constructs its own
  `CategoryBudgetsDao(db)`.

→ `categories` parents first.

  returns only that category's.
