# M0-T06 — Verify the CI workflow

- **Milestone:** M0
- **Dependencies:** M0-T05
- **Estimated new code:** none (configuration verification)

## Goal
Confirm `.github/workflows/ci.yml` runs green on a real pull request.

## Context — read first
- `docs/08-ci-cd.md`

## Steps
1. Push the repository to a GitHub remote (create one if it does not exist).
2. Open a pull request from a feature branch into `main`.
3. Confirm the `CI` workflow runs and every step passes: `pub get`,
   `build_runner`, format check, `analyze`, `test`.
4. Configure branch protection on `main` per `docs/08-ci-cd.md`: require the
   `CI` check, disallow direct pushes.

## Acceptance criteria
- [ ] The `CI` check is green on a pull request.
- [ ] Branch protection on `main` requires `CI` to pass.
- [ ] M0-T06 is ticked in `docs/06-task-breakdown.md`.
