## Public API to produce
```dart
// lib/domain/budgeting/invariants.dart
/// Invariant 1 (`docs/04`): with no credit overspending, the sum of every
/// category's `available` plus Ready-to-Assign equals the total working
/// balance of all on-budget accounts. Returns null when it holds, else a
/// human-readable message.
String? checkBudgetEquation({
  required MonthBudget monthBudget,
  required Money onBudgetWorkingTotal,
});

/// Invariant 2: a split transaction's `amount` equals the sum of its
/// sub-transactions' `amount`s. Returns null when it holds, else a message.
String? checkSplitSum({
  required Transaction parent,
  required Iterable<SubTransaction> subTransactions,
});

/// Invariant 3: a transfer's two transaction rows hold equal and opposite
/// `amount`s. Returns null when it holds, else a message.
String? checkTransferPair(Transaction a, Transaction b);
```

## Implementation notes
- Each function returns `null` on success and a descriptive `String` (naming the
  offending ids and the two amounts) on failure ‚Äî callers (tests, and a future
  debug assertion) collect non-null results.
- **`checkBudgetEquation`**: sum `monthBudget.lines` `available`, add
  `monthBudget.readyToAssign`, compare with `onBudgetWorkingTotal`. The caller
  supplies `onBudgetWorkingTotal` as the sum of `computeAccountBalances(...)
  .working` over on-budget accounts (M2-T02). The identity holds only when no
  category is credit-overspent ‚Äî document that precondition in the doc comment.
- **`checkSplitSum`**: only meaningful when `parent.isSplit`; sum the
  non-deleted `subTransactions` and compare with `parent.amount`.
- **`checkTransferPair`**: assert `a.amount == -b.amount` and that each row
  points at the other (`a.transferTransactionId == b.id` and vice versa).
- `Money` arithmetic only ‚Äî no `double`. Invariant 4 ("assigned may be
  negative") is a permission, not a runtime check; invariant 5 ("no double") is
  enforced by the analyzer ‚Äî neither needs a function here.

## Tests to write
File: `test/domain/budgeting/invariants_test.dart`
- **Unit**: each check returns `null` for a valid input and a non-null message
  for a deliberately broken one (a split whose subs do not sum; a transfer pair
  with unequal amounts).
- **Property-style** (`docs/07`): with a seeded `Random`, generate many budgets
  via `BudgetFixture` ‚Äî random on-budget accounts, categories, assignments, and
  non-credit transactions ‚Äî run `BudgetService.computeMonth`, and assert
  `checkBudgetEquation` returns `null` every iteration (‚â• 50 iterations). Use a
  fixed seed so failures reproduce.
- A property run that includes split transactions asserts `checkSplitSum`
  passes for every split.
- A property run that includes transfer pairs asserts `checkTransferPair`
  passes for every pair.
