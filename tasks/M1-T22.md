## Public API to produce
```dart
// lib/features/payees/domain/payee.dart
@freezed
class Payee with _$Payee {
  const factory Payee({
    required String id,
    required String budgetId,
    required String name,
    String? defaultCategoryId,
    String? transferAccountId,
  }) = _Payee;

  const Payee._();

  /// True when this payee stands for a transfer to/from an account.
  bool get isTransferPayee => transferAccountId != null;
}

// lib/features/payees/domain/payee_repository.dart
abstract interface class PayeeRepository {
  Stream<List<Payee>> watchAll(String budgetId);
  Future<Payee?> findById(String id);
  Future<void> save(Payee payee);
  Future<void> delete(String id);
}

// lib/data/daos/payees_dao.dart
@DriftAccessor(tables: [Payees])
class PayeesDao extends DatabaseAccessor<AppDatabase> with _$PayeesDaoMixin {
  PayeesDao(super.db);
  Stream<List<PayeeRow>> watchByBudget(String budgetId);
  Future<PayeeRow?> findById(String id);
  Future<void> upsert(PayeesCompanion payee);
  Future<void> deleteById(String id);
}

// lib/features/payees/data/payee_mappers.dart
Payee payeeFromRow(PayeeRow row);
PayeesCompanion payeeToCompanion(Payee payee);

// lib/features/payees/data/payee_repository_impl.dart
class PayeeRepositoryImpl implements PayeeRepository {
  PayeeRepositoryImpl(AppDatabase db);   // constructs its own PayeesDao(db)
}
```

## Implementation notes
- `payees` has **no** `created_at` / `updated_at` columns (`docs/03`), so
  `payeeToCompanion` takes no timestamp arguments and `save` needs no
  preserve-`createdAt` logic — it is a plain `insertOnConflictUpdate`.
- `watchAll` orders by `name`; map the DAO stream with `rows.map(payeeFromRow)`.
- `save` has upsert semantics. `delete` is a hard delete (payees are not
  soft-deleted — `docs/03`).
- `Payee` needs the private `const Payee._();` constructor for the
  `isTransferPayee` getter.
- The impl lives in `data/`; the domain entity and interface stay Drift-free.
